HTML Document Structure

Before a webpage can be rendered properly on a browser, the elements on the page need to follow a certain structure the browser parser understands. This tutorial will give a quick overview of how a web document needs to be structured for proper rendering across browsers. It's a kind of a template you can copy and paste on your text editor or IDE and go on to add your web content. It is meant for the newbie in web design.

What You Need to Build Your First Web Page

You need the following to create your first web page:

  1. A PC or Mac to work on (obviously)
  2. Text editor, such as Notepad for Windows or textEdit for Mac to type your scripts
  3. A browser such as Chrome or Safari to view your web pages
  4. Knowledge of HTML
  5. Knowledge of CSS
  6. Knowledge of JavaScript (at the moment not mandatory)

You don't need to install anything in your PC or Mac to display your first webpage. Notepad come shipped with Windows and textEdit come shipped with a Mac OS X. However, installing a professional editor such as Sublime, Visual Studio or Notepad++ can go along way in making your coding environment comfortable. HTML, CSS and JavaScript is knowledge you will acquire by and by.

A webpage is an HTML document that can be interpreted by a web browser such as Chrome, Firefox or Safari and display its contents. The two scripting languages that are fundamental to building webpages are HTML and CSS. HTML stands for hypertext markup language and CSS is an acronym for Cascading Style Sheets. HTML is used to markup the content to be displayed on the brower while CSS describes how the content should be presented to the user on the screen. This involves page layout, text formats such as color, size and positioning of page elements.

HTML and CSS help build static web pages. To add interactivity to your webpages, you need to add JavaScript to your knowledge base. There are many libraries and frameworks associated with these languages, but solid understanding of these three languages enables one work well with their frameworks and libraries. This tutorial is the first of an intended web development series and we are starting with HTML document structure.

Semantic HTML Markup

Apart from structuring web documents for proper rendering, modern web documents need to follow SEO (search engine optimization) markup principles. This not only enables search engines to crawl and index the webpages so that they can appear on search page results, but also aid user agents and screen readers to understand the pages. Here we will follow SEO principles.

Semantic HTML is the use of HTML markup to reinforce the meaning (semantic) of the web contnent rather than its presentation.

HTML uses tags to markup the elements to be presented by the browser. A tag consists of the element name enclosed by angle brackets < and >. For example to draw a horizontal line on the page we use <hr> tag. Some of these tags must have their closing equivalent and others don't. For example the html tag has the openning <html tag and </html> closing tag. The <hr> doesn't have a closing tag. We are going to learn more HTML tags as we proceed the same way we learn English vocabulary to build language mastery.

How to Structure an HTML Document

A web document starts with <!DOCTYPE html>. This element tells the brower that the document it is about to parse is an HTML document. Other markup languages browsers can parse are xhtml or xml. Here we follow the HTML standards, more specifically HTML5. By convention HTML elements and their attributes are typed in lowercase.

<!DOCTYPE html> does not have a closing tag. Therefore, what follows is the <html> tag. All elements of the web document must be placed within the <html lang="en" > openning tag and </html> closing tag. In other words, this is how it will look like:

<!DOCTYPE html>
<html lang="en">
		
	.... the rest go here ...
	
</html>

This tells the brower where the HTML document start and where it ends.

The <head> Section

A proper web document has a section marked by <head> start tag and </head> end tag. Head elements apart from <title> element, are not rendered on the screen. However, they relay important information to the browser and search engines when they crawl the page. Most of the element in the </head> tags are non-closing <meta> tags. We are going to add the most important <meta> tags in the head section. 

<!DOCTYPE html>
<html lang="en">
	<head>
		<meta charset="utf-8">
		<meta name="viewport" content="width=device-width, initial-scale=1.0"> 
		<meta name="author" content="your name">
		<meta name="description" content="Briefly describe your web page her ">
		<meta name="keywords" content="type keywords/ phrases that searchers may use to search for your contents.">
		<title>HTML Document Structure</title>
	</head>		
</html>

Let's explain their usage before we move to the next section. utf-8 tells the brower to use 8-bit Unicode Transformation Format to parse the document. The meta viewport prepares the browser to render the document using responsive layout. SEO principles requires that webpages be designed with responsive layout. That means the same webpage can be viewed nicely on large screens and smartphone screens without the user having to scroll sideways to view page items.

Responsive web design is nowadays mandatory for SEO.

Use the meta author to put your name to show that you are the one who designed the web page. meta description is important because its content appear on search enginge result page as the description of the web page.

The meta keywords contain keywords and phrases that you have used in your document and are likely to be the ones people will use on search engines to look up content like yours. Both description and keywords used to be important parameters search engines ranked webpages in their indices till developers started abusing them through keywords stuffing. Nowadays search engines use sophisticated algorithms that utilize various parameters to rank webpages.

Keyword stuffing is loading a webpage with the same target term in an attempt to gain unfair rank advantage in search engine. Keyword stuffing can lead to a website penalized or banned by search engines.

The <title> tag is only tag in the <head> section whose contents will render on the browser as the title of the webpage. The <title>, <description> and <keywords> tags need to be consistent with the general content of the webpage for the page to fair well with search engines, otherwise search engines will penalize the page.

The <body> Section

What follows after the <head> section is the <body> secton. All the content that will display on the brower screen must be placed within the openning <body> tag and ending </body> tag. The body section itself must be placed within the html tags. So let's add the body section to the document.

<!DOCTYPE html>
<html lang="en">
	<head>
		<meta charset="utf-8">
		<meta name="viewport" content="width=device-width, initial-scale=1.0"> 
		<meta name="author" content="your name">
		<meta name="description" content="Briefly describe your web page her ">
		<meta name="keywords" content="type keywords/ phrases that searchers may use to search for your contents.">
		<title>HTML Document Structure</title>
		<link rel="stylesheet" href="">
	</head>
	<body>
	...All visible page content will go here...
	</body>
</html>

The <body> section most times is subdivied sections for layout and semantic purposes. The <div> and <span> tags are used extensively to section the page layout and positioning. However, following SEO design principles, we also need to use semantic tags to portray the meaning rather than presentation only. Let's look at the ones you are likely to use in most of your webpages.

The <header> section

The <header> section is normally placed at the upper section of the webpage. It can encompass navigation links , company name, logo or the heading of the page. Navigation links are in turn placed in <nav> tags. Accoding to SEO principles each web page should to have only one <h1> heading element. Any other headings on the page should be given lower heading tags (such as <h2>, <h3> and so on) to signify sub-headings.

The <main> Section

The <main> element represent the dominant content of the <body>. There is supposed to be only one <main> on a webpage. Since this section in most times is the largest of all it can be subdivided by other semantic element tags such as <aside>, <section> or <article> as the developer sees it fit. Later on, these sections can be styled with CSS to display appropriately on different screen sizes.

The <article> Section

The <article> element denotes a self contained composition of the page that can be distributed independently such as articles of a newspaper. Each <article> typically should be identified by a heading; either <h1> or <h2> tag. A web document can have multiple <article> sections within a <main> section.

The <section> Section

<section> element surround generic section of a webpage or app such as chapters, headers, footers .It can be used for a standalone sections which do not have more specific semantic to represent them.

The <aside> Section

It represents a portion a document whose content is not directly related document's main content. <aside> element is frequently used to represent sidebars.

The <footer> Section

Footer section is placed at the bottom of the document, just before the </body> closing tag. The <footer> can contain page navigation links, but most importantly, should contain information about page content that does not change across the site. Information such as contact address, organisation name, copyright should be placed here.

Let's use these sections in a most likely way you may want to use them.

The above sectioning may be used for a two column page where the <aside> is a smaller column floated to the left or right. The main column designated by <main> with various sectioning.
