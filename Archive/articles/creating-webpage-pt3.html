Creating a Webpage: Designing a Web Form

Web forms are common on webpages. They are a means of interacting with web users and help in collecting views or information from them.

In this installment we are going to create a web form for placing reservation at our in progress Mara Resort webpage. At the end of this tutorial you should be able to:

  • create a web form and determine what appropriate form input types to deploy.
  • control how form elements function using attributes.
  • style a web form with CSS.

The <Form> Element

A web form starts with <form> and ends with </form> tags. In between the two tags goes the various input fields and other form elements.

<form>
    ...input elements go here...
</form>

Form <input> Types

Every form will have at least one <input> element. How the input will function depends on its type attribute. If the type attribute is not specified in the <input> element, then the input will default to <input type="text">.

There are more than a dozen form control types we can't exhaust them here, and there is no way you will need all of them at once.

Usage syntax is: <input type="value">

Here is a list of input types you are likely to come across with their brief explanation.

Type valueDescription
buttonDefines a clickable button with no default behavior (mostly used with JavaScript).
checkboxA check box allows its value to be checked (selected) or unchecked (deselected).
fileAn input field that lets the user select a file from local storage for upload.
hiddenDefines an input field that is hidden from view but whose value is submitted with the form.
imageDefines a graphical submit button. The displayed image is defined by the src attribute.
passwordDefines a single-line text field whose value is obscured.
radioA radio button allows a single value to be selected out of multiple choices with the same name value.
resetA button that clears the form contents to default values (its usage is nowadays discouraged).
submitA button that submits the form.
text Defines a single-line text field.

Introduced in HTML5

colorA control that opens a color picker when active in supporting browsers.
dateA control that opens a date picker or numeric wheels for entering date, year, month, and day (no time).
datetime-localA control for entering a date and time, with no time zone.
emailA field that looks like a text input but only allows valid email addresses.
monthA control for entering a month and year, with no time zone.
numberAn input field for entering a number. Displays a spinner and adds default validation when supported.

For example, you can use email type attribute in the following way:

<form>
    Email: <input type="email">
</form>

The above code will result in a text field that allows only properly constructed email addresses.

Other <input> Attributes

Apart from the type attribute, an input can have other attributes that can be used to control the behavior of the form. I have tabulated some of them below.

AttributeDescription
autocomplete Hint for field autofill feature
autofocusAutomatically focus the form control when the page is loaded
checkedWhether the control is selected. Applies to radio and checkboxes.
disabledWhen used, the form control is disabled.
maxlengthMaximum length (number of characters) of the value. Applies to password, search, tel, text, url input types.
minlengthMinimum length (number of characters) of the value. Applies to password, search, tel, text, url input types.
multipleWhether to allow multiple values.
nameNames the form control. It's name is submitted with the form as part of a name/value pair.
patternA validation pattern the value of an input field must match to be valid.
placeholderText that appears in an input field when it has no value set. Applies to password, search, tel, text, url input types.
readonlyMakes the value in an input control not editable.
requiredA value is required or must be checked for the form to be submittable.
sizeThe width of the control in characters. It applies to email, password, tel and text input types.
typeType of form control
valueThe current value of the form control submitted with the form as part of a name/value pair.

An example of using some of these attributes is like:

<form>
    Email: <input type="email" name="email" placeholder="Your email address" size="20" maxlength="30" autocomplete="off" required>
</form>

Other Form Controls

Other form controls that do not use <input type=" "> are textarea and select. <textarea> control allows multi-line text and can be used to collect user views or comments. <select> provides a list of options for the user to choose from.

Let me explain some of the not-so-obvious attributes we have used in the above form.

Form Action Attribute

In the above code we have used action="ContactForm.php" in the opening form tag. Normally, the value of the action=" " attribute specifies the file that will receive or process the form values on form submission. In our case, ContactForm.php is the file that will process the form when submitted. This is a PHP server-side file.

There are other ways to process form values than specifying a file using action attribute. So you are likely to see forms that don't use action attribute.

Form Method Attribute

The method attribute in the form element tells the browser how to send the form data to the server. Its value specifies the HTTP method to be used in submitting the data. There are two HTTP methods for submitting forms: GET and POST.

GET Method

On form submission GET method appends form data at the end of the page URL. Consequently, the form values will be visible in the page URL at submission and can be cached by the browser. GETalso limits the size of data it can send. The whole URL including the data should not be more than 2048 characters. Using GET method is unsecure way of submitting data to a server.

Never submit sensitive data using GET method. Nonetheless, there are times GET is the most ideal to use. For example a search form should use GET method so that users can bookmark the search results.

POST Method

POST method appends form data in the body of the HTTP request. This method has no size limitation, neither can one bookmark the result. Information sent using POST method is not visible to the user. We have used method="POST" in the form above because the data we are collecting is confidential.

POST method is ideal for submitting sensitive data securely to the server, especially if it is used over HTTPS protocol.

Radio Buttons

<input type="radio"> is used when the user has to choose one out of two or three options. The group of radio buttons should share the same name (name="gender" above) but different values (value="male", value="female", value="other" above). For instance when the form above is submitted, the processing file or program will receive gender = "value of option selected"

Pattern Attribute

Pattern attribute controls what characters or how data should be entered in the field. For example in the tel field above, we have used pattern="/^(\+[\d]{,3})?[ .-]?[\d]{3}[ .-]?[\d]{3}[ .-]?[\d]{3}$/". This allows an optional country code of upto three digits, then mandatory nine digits in group of threes, seperated by a space, period or hyphen. The placeholder="(+123)-555-555-555" tries to guide the user how to fill the telephone number.

Form <select> Control

<select> element is used to output a list of options a user can select from. The items choices are placed within <option> tags with a value attribute. In our case, I have listed some East African countries and major tourist markets for The Mara Resort.

Conclusion

Most of the time we use a small set of HTML form controls, but sometimes you may need the ones not commonly used. Knowing how to design a form is one thing, making it deliver secure data to the server is another. You should always ensure form data is validated before it can be processed by the server. Supplementally, you can check out HTML5 Form Validation and Form Validation in JavaScript for more understanding on how to secure forms.
