HTML Document Structure

Before a webpage can be rendered properly on a browser, the elements on the page need to follow a certain structure the browser parser understands. Here, I will give a quick overview of how a web document needs to be structured for proper rendering across browsers. It's kind of a template you can copy and paste on your text editor or IDE and go on to add your web content. This is meant for the newbie in web design.

What You Need to Build Your First Web Page

You need the following to create your first web page:

  1. A PC or Mac to work on (obviously)
  2. Text editor, such as Notepad for Windows or textEdit for Mac to type your scripts
  3. A browser such as Chrome or Safari to view your web pages
  4. Knowledge of HTML
  5. Knowledge of CSS
  6. Knowledge of JavaScript (at the moment not mandatory)

You don't need to install anything in your PC or Mac to display your first webpage. Notepad come shipped with Windows and textEdit come shipped with a Mac OS X. However, installing a professional editor such as Sublime, Visual Studio or Notepad++ can go along way in making your coding environment comfortable. HTML, CSS and JavaScript is knowledge you will acquire by and by.

A webpage is an HTML document that can be interpreted by a web browser such as Chrome, Firefox or Safari and display its contents. The two scripting languages that are fundamental to building webpages are HTML and CSS. HTML stands for hypertext markup language and CSS is an acronym for Cascading Style Sheets. HTML is used to markup the content to be displayed on the browser while CSS describes how the content should be displayed to the user on the screen. This involves rendering page layout, text formats such as color, size, positioning of page elements and so on.

HTML and CSS help build static web pages. To add interactivity to your webpages, you need to add JavaScript to your knowledge base. There are many libraries and frameworks associated with these languages, but solid understanding of these three languages enables you work well with their frameworks and libraries. This tutorial is the first of an intended web development series and we are starting with HTML document structure.

Semantic HTML Markup

Apart from structuring web documents for proper rendering, modern webpages need to conform to SEO (search engine optimization) markup principles. This not only enables search engines to crawl and index the webpages, so that they can appear on search page results, but also aid user agents and screen readers to understand the pages. Here, we will try our utmost level to conform to SEO principles.

Semantic HTML is the use of HTML markup to reinforce the meaning (semantic) of the web contnent rather than its presentation.

HTML uses tags to markup the elements to be parsed by the browser. A tag consists of the element name enclosed by angle brackets < and >. For example, to draw a horizontal line on the page we use <hr> tag. Some of these tags must have their closing equivalent and others don't. The html tag for example, has the opening <html> tag and the closing </html> tag. The <hr> doesn't have a closing tag. We will learn more HTML tags as we proceed, the same way we learn English vocabulary to build language mastery.

How to Structure an HTML Document

A web document starts with <!DOCTYPE html>. This element tells the browser that the document it is about to parse is an HTML document. Other markup languages browsers can parse are xhtml or xml. In this series we are using the HTML standards, and more specifically HTML5.

By convention HTML elements and their associated attributes are typed in lowercase.

Since the <!DOCTYPE html> does not have a closing tag, the element that follows it in the document is an <html> tag. All elements in the web document must be placed within the <html lang="en" > opening tag and </html> closing tag. You will notice that we have used an attribute lang="en" in the opening <html> tag. We are telling the browser that the language used in the page is English. This is how the structure looks like so far:

<!DOCTYPE html>
<html lang="en">
		
	.... the rest go here ...
	
</html>

The above HTML tags tell the browser where the page starts and where it ends.

The <head> Section

A proper web document has a section marked by <head> start tag and </head> end tag. Head elements except the <title> element, are not rendered on the screen. However, they relay important information to the browser and search engines when they crawl the page. Most of the elements in the head section are non-closing <meta> tags. Let's add the most used <meta> tags in the head section. 

<!DOCTYPE html>
<html lang="en">
	<head>
		<meta charset="utf-8">
		<meta name="viewport" content="width=device-width, initial-scale=1.0"> 
		<meta name="author" content="your name">
		<meta name="description" content="Briefly describe your web page her ">
		<meta name="keywords" content="type keywords/ phrases that searchers may use to search for your contents.">
		<title>HTML Document Structure</title>
	</head>		
</html>

Let us explain the usage of each meta tag before we move to the next section. utf-8 tells the browser to use 8-bit Unicode Transformation Format to parse the document. The meta viewport prepares the browser to render the document using responsive layout. SEO principles requires that webpages be designed with responsive layout. That means the same webpage can be viewed nicely on large screens and mobile devices without the user having to scroll sideways to view page items.

Responsive web design is nowadays mandatory for SEO.

Use the meta author to add your name as the one who designed the web page. meta description is important because its content appear on search enginges' result pages as the description of the webpage.

The meta keywords contain keywords and phrases that you have used in your document and are likely to be the ones people will use on search engines to look up for content like yours.

Both description and keywords used to be important parameters search engines used to rank webpages in their indices, till developers started abusing them through keywords stuffing. Nowadays search engines use sophisticated algorithms that utilize various parameters to rank webpages.

Keyword stuffing is loading a webpage with the same target term in an attempt to gain unfair rank advantage in search engines. Keyword stuffing can lead to a website being penalized or banned by search engines.

The <title> tag is the only tag in the <head> section whose contents will render on the browser as the title of the webpage. The <title>, <description> and <keywords> tags need to be consistent with the general content of the webpage for the page to fair well with search engines, otherwise, search engines may penalize the page.

The <body> Section

What follows after the <head> section is the <body> secton. All the content that will display on the browser's screen must be placed within the opening <body> tag and closing </body> tag. The body section itself must be placed within the html tags. So let's add the body section to the document.

<!DOCTYPE html>
<html lang="en">
	<head>
		<meta charset="utf-8">
		<meta name="viewport" content="width=device-width, initial-scale=1.0"> 
		<meta name="author" content="your name">
		<meta name="description" content="Briefly describe your web page her ">
		<meta name="keywords" content="type keywords/ phrases that searchers may use to search for your contents.">
		<title>HTML Document Structure</title>
	</head>
	<body>
	...All visible page content goes here...
	</body>
</html>

Most times the <body> section is subdivided for layout and semantic purposes. A <div> tag can be used to section a webpage for layout and positioning. However, to abide by SEO design principles, we also need to use semantic tags to designate the meaning of the sections rather than presentation only. Let's use the semantic tags you will likely use in most of your web projects.

The <header> section

The <header> is a section normally placed at the upper section of the webpage. It can encompass navigation links , company name, logo or possibly, the main heading of the page. Navigation links are in turn placed in <nav> tags.

Accoding to SEO principles each webpage should have only one <h1> heading element. Any other heading on the page should be given lower heading tag such as <h2>, <h3> and so on.

The <main> Section

The <main> element represent the dominant content of the <body>. Since this section in most cases encompasses many components, it can be subdivided further by other semantic tags such as <aside>, <section> or <article> as the developer may see it fit. Later on, these sections can be styled with CSS to display appropriately on different screen sizes.

There is supposed to be only one <main> element on a webpage.

The <article> Section

The <article> element denotes a self contained composition of the page that can be distributed independently such as articles of a newspaper. Each <article> typically should be identified by a heading; either <h1> or <h2> tag. A web document can have multiple <article> sections within a <main> section.

The <section> Section

<section> element surround a generic section of a webpage or app such as chapters, headers, footers. It can be used for a standalone section which does not have a more specific semantic to represent it.

The <aside> Section

It represents a portion of a document whose content is not directly related to the main content. <aside> element is frequently used to represent sidebars on large screens and hid or styled as block element in smaller screens.

The <footer> Section

<footer> section is placed at the bottom of the document, just before the </body> closing tag. The <footer> can contain page navigation links, but most importantly, should contain page content that does not change across the site. Content such as contact address or copyright information should be placed in this section.

The above sectioning may be used for a two column page, where the <aside> is a smaller column floated to the left or right. The main column containing <section> with two <article> sections. Most of these semantic tags we have used were introduced in HTML5.

Although <div> tags could have been used to section the page as done above, they have no semantic value and thus would be violating SEO principles.
