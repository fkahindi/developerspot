Block-level Vs Inline Elements

HTML elements can be categorized as either block-level or inline elements. Block-level element occupy the entire width of the parent container. By default browsers display block elements in a new line as a block. Examples of block-level elements are <p>, <div>, <h1> ... <h6>, <header>, <footer> and many others.

On the other hand inline-level elements only occupies space bounded by the tags and don't break the flow of the content they are placed in. By default they don't start on a new line. Examples of inline-level elements are <span>, <a>, <br>, <img>, <button> and so on.

The Paragraph <p> Element

Paragraphs in a web document are created by enclosing the text in <p> and </p> tags. The <p> is both a semantic and presentational markup. As I have already said, it is also a block element. Most likely you will use alot of this element in your web projects.

The Content Division <div> Element

The <div> is a container for flow of content. It does not inherently represent anything other than grouping content so that it can be styled together. It has no semantic value. Use <div> when no other sectioning semantic element is appropriate to use. Normally, attributes such as class or id are used in the opennnig <div> tag to target the <div> section for styling.
