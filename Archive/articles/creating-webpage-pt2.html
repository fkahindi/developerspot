How to Design a Responsive Webpage with Collapsible Menu

This is a second part of a series on how to Design a Webpage. In the first part, we learned how to structure an HTML document using both semantic and presentational HTML elements. In that post we defined these elements and why we use them.

In this article we are going to add content to the document structure and design a responsive page layout with collapsible navigation menu. That means, on smaller devices the menu bar will collapse to a hamburger icon ☰. We will do this, using HTML and Cascading Style Sheets (CSS). We will also enable the show/hide menu content, with a small JavaScript code.

If you want to check out the first part, use this link, how to structure a web page. At the end of this tutorial you should be able to design:

  • A responsive webpage with the layout you prefer.
  • A collapsible navigation menu.

In the process you would have learnt how to apply:

  1. Unordered lists using <ul> and <li> in menus.
  2. CSS selectors: class, id,element and pseudo element selectors to target page elements for styling.
  3. CSS @media queries for responsive web design.

A responsive webpage is a page designed to look good in all screen sizes: big screens, desktops, laptops, tablets and mobile phones. Such a page adjusts automatically to fit the screen viewport width of the device.

The article assumes the reader is a beginner in HTML stuff. So, before we dive in, let's explain some HTML concepts, that will help in designing a responsive webpage layout.

Block vs Inline Elements

HTML elements can be categorized as either block or inline elements. A block element occupies the entire width of the parent container. By default, browsers display block elements in a new line, having height and width dimensions.

Examples of block-level elements are <p>, <div>, <h1> ... <h6>, <header>, <footer> and many others.

On the other hand, an inline element only occupies the space bounded by its tags and does not break the flow of content. By default, they don't start on a new line. Examples of inline elements are <span>, <a>, <br>, <img>, <button> and so on.

The Paragraph <p> Element

Paragraphs in a web document are created by enclosing the text between <p> and </p> tags. The <p> is both a semantic and presentational markup. As I have already said, it is also a block element. Most likely you will use alot of this element in your web projects.

The Content Division <div> Element

The <div> is a container of flow of content. It does not inherently represent anything other than grouping content together for styling. In other words, it has no semantic value. Use <div> when no other sectioning semantic element is appropriate to use. Most times, selectors such as class and id are used in the opennnig <div> tag to target the div for styling.

Cascading style Sheets

CSS rules are used to style html elements for presentation. There are three areas style rules can be placed on a web document, where the browser can read them:

  1. Inline styles: they are placed in the opeining tag of the element to be styled. For example <h3 style="font-size:20px; color:orange;"> Heading <h3> makes the "Heading" 20 pixels in size and orange in color.
  2. Internal styles: are placed at the <head> section of the html document, and are enclosed by <style> and </style> tags. Styles at the head of a document apply to all targeted elements in the document.
  3. External styles: are placed in a seperate external file. The filename needs to be saved with an extension .css. Any page that needs to use the styles has to reference the file in the <head> section of the html document. The syntax of the reference should be in the form of: <link rel="stylesheet" href="path-to-file/file-name.css">

Inline styles are generally discouraged, because they make website maintenance difficult and go against best practises of seperating content from design.

For simplicity, we are going to place our styles in the <head> section of our document.

CSS Box Model

CSS, by default treats a block element as a rectangular box, with height and width. In the box model, the total width and height of the block element is made up of the sizes of the margin, border, padding and the content of the element. The figure below illustrates what I mean.

The outer dash line in the above figure, defines the actual width of the element.

For instance, setting a div width to width:100px, border-width:3px, margin:10px and padding:5px makes the div occupy a total width of 118px. That is, 100 + 3 + 10 + 5 = 118

However, when box-sizing:border-box rule is applied to a block element, the box model of the element changes, so that the width and height of the padding and border no longer adds to the size of the element. Instead, it constrains the content of the element itself. The figure below illustrates what I mean.

That means, if the <div> is set to width:100px, padding:5px, border-width:3px and margin:10px, the div will need a window of width 110px (100 + 10) to display without having to scroll side ways. The border-width: and padding: will instead contrain the content inward than adding to the div's width. I hope that's clear.

Paste the following style rule at the beginning of your styles:

*{
    box-sizing:border-box;
}

The * is a wildcard selector. The above rule will be applied to all block elements.

Still on block elements: we can force block elements display as inline elements, that is, placed side by side. The rule to be applied on the element is display:inline-block. The sum of the elements' width, however, must not exceed the viewport width.

Styling a Responsive Layout

Next, we want the webpage layout be two-column on wide screens, such as desktops, laptops and tablets in landscape. One of the columns (aside section) will occupy 35% of the available space and the other column (main section) will take 55% of the remaining space. The header and footer will display in single column layout, in all devices.

On smaller screens, like mobile phones and tablets in potrait, the page will have one column layout. To accomplish what we are saying, we will use media queries.

Media Queries

A media query, is a CSS technique that enables the browser to query the client media viewport for specific width breakpoints. For each breakpoint, the browser applies the appropriate styles.

In our case, we will make it simple. We will use a single media query for divices with viewport width of 768px or less and general rules for devices with viewport width greater than 768px.

So, we will write our media query as @media screen and (max-width:768px){ }. This rule will query the browser screen width at the time of page rendering to determine if it is 768px or less. If it is so, the browser will apply the rules defined within the braces { }.

In our layout, the content area of the page will take 80% of the viewport on big screens ( with aside content on the left and the main content on the right). However, on small devices (with viewport width of 768px or less), the content will cover 100% of the viewport.

We will use display:flex model to achieve the layout we want.

Earlier, before this update I had used float to setup a two column layout. Never use float again for laying out a page. flex is now supported by all major browsers and was designed for that purpose.

The following styles should give us the layout we want.

/* Page layout  */        
.header-content {
    max-width: 70%;
    margin: 20px auto;
}
.container,
header,
footer {
    width: 80%;
    margin: 0 auto;
    padding: 10px;
}
.container {
    display: flex;
    flex-direction: row;
    justify-content: space-between;
}
aside {
    width: 35%
}
main {
    width: 55%
}
.trade-mark {
    font-family: Allura;
    font-size: 2em;
    text-align: center;
    padding: 0;
}
footer {
    text-align: center;
}
@media screen and (max-width: 768px) {
    /* for tablets in portrait and mobile phones */
    .container,
    header,
    footer {
        width: 100%;
    }
    main,
    aside {
        width: 95%;
    }
    .container {
        flex-direction: column-reverse;
    }
    .header-content {
        max-width: 80%;
        margin: 10px auto;
    }
}

The flex-direction: row makes the aside and main laid side by side, (row). The flex-direction: column-reverse makes the two section laid on top of each other, (column) with their order reversed.

When choosing fonts for a webpage, it is recommended to use safe web fonts. These are fonts that are common in many client systems. However, sometimes you would want a unique font face for its feel, and want your viewers to get the same experience, irrespective of its availability in their systems. In such cases, you have two options: either include the font files with the styles or use content delivery network (CDN) that has the fonts.

For us, we will include the font files with our stylesheet. So, create a fonts folder in your mara-project folder. Then search and download the two font files from the web, saving them in the fonts folder.

We are going to include the font files at the beginning of the stylesheet, with @font-face{} style rules. That is:

@font-face{
font-family:Allura;
src: url("fonts/Allura-Regular.ttf");
font-display: swap;
}
@font-face{
font-family: Aclonica;
src: url("fonts/Aclonica.ttf");
font-display: swap;
}

It's important to get the file path and filename extension right. In our case both files are true type fonts, .ttf. Another issue is to read the copyright licenses that come with web fonts. Some web fonts may only be free, if used for non-commercial purposes.

We have used font-display:swap rule to enable the browser draw the text immediately with a fallback font, till the web font is loaded, then swap them.

At the moment we can't toggle the menu, ☰ icon to display or hide the menu items. To be able to do that we need a small code in JavaScript.

The code will listen to click events on the buttons we created. It will also listen to browser window resizing event and determine whether the media query breakpoint is reached and apply the appropriate styles.

Copy the JavaScript code below and paste it at the bottom of the document before the </body> ending tag.

<script>
    /* Assign target elements to variables */
    let navContent = document.getElementById('nav-content');
    let dropdownNavBtn = document.getElementById('dropdown-nav-btn');
    let closeBtn = document.getElementById('close-btn');    
        /* listen to click events on dropdown-nav-btn and 
            toggle target elements appropriately 
        */
    dropdownNavBtn.onclick = function() {
            this.style.display = 'none';
            navContent.style.display = 'block';
            closeBtn.style.display = 'block';
        }
        /* listen to click events on close-btn and 
            toggle target elements appropriately 
        */
    closeBtn.onclick = function() {
        this.style.display = 'none';
        navContent.style.display = 'none';
        dropdownNavBtn.style.display = 'block';
    }
    /* listen to window resize event and toggle elements appriately */
    window.onresize = function() {
        if (window.innerWidth < 768) {
            dropdownNavBtn.style.display = 'block';
            navContent.style.display = 'none';
            closeBtn.style.display = 'none';
        } else {
            navContent.style.display = 'block';
            dropdownNavBtn.style.display = 'none';
            closeBtn.style.display = 'none';
        }
    }
</script>

Save the changes and refresh the browser. You should now be able to toggle the menu. When you click the ☰ icon, the menu content will be displayed. At the same time the close, × button will appear. When you click the close button, the menu items will be hid and the hamburger, ☰ icon will show up.
