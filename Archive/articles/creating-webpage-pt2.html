A responsive web page is designed to look good in all devices: desktops, laptops and phones. Such a page adjusts automatically, to fit the screen width it is being viewed.

We will be dropping in new HTML tags along the way that will become part of your arsenal. But before we dive in, some important info about HTML elements.

Block-level vs Inline Elements

HTML elements can be categorized as either block-level or inline elements. A block-level element occupies the entire width of the parent container. By default browsers display block elements in a new line as a block. Examples of block-level elements are <p>, <div>, <h1>,...,<h6>, <header>, <footer> and many others.

On the other hand, an inline-level element only occupies the space bounded by the tags and does not break the flow of the content they are placed in. By default they don't start on a new line. Examples of inline-level elements are <span>, <a>, <br>, <img>, <button> and so on.

The Paragraph <p> Element

Paragraphs in a web document are created by enclosing the text in <p> and </p> tags. The <p> is both a semantic and presentational markup. As I have already said, it is also a block element. Most likely you will use alot of this element in your web projects.

The Content Division <div> Element

The <div> is a container for flow of content. It does not inherently represent anything other than grouping content so that it can be styled together. It has no semantic value. Use <div> when no other sectioning semantic element is appropriate to use. Normally, selectors such as class or id are used in the opennnig <div> tag to target the <div> section for styling.

Now we are going to style the page and also make it responsive. Style rules can simply be placed on the head section or in an external files.

There are three places style rules can be placed:
1. Inline styles within elements. They are discouraged because it makes it harder to debug the HTML document.
2. Head styles these are styles placed within the head section of the document. Head styles are good and leads to faster page loading but not ideal when the rules are many or need to be used in more than one page.
3. External styles, are styles placed in an external file or files. They are ideal because one single file can be used across multiple pages or entire website.

Styling A Webpage

You style web elements by targeting them with selectors, then define rules on how they should be presented on the browser. Let's start by targeting the body of the page. The body defines the visible area of the web page. We are going to style it thus:

<style>
body {
    font-family: sans-serif, Helvetica;
    font-size:16px;
    color: #333;
    background-color: #C2B280;
    margin: 0; 
}
</style>

The above styles sets the default font-type to sans-serif, with an alternative Helvetica font if the first font is not available. Base font-size is set to 16 pixels. Primary text color is set to a shade of black and Background color to sand but represented in hexadecimal numbers. Margin is set to 0 all round.

Refresh the page and see how the browser has applied the style rules.

Next we are going to set the width of the header, footer and main sections to 100%, margin and padding set to 0.

header, footer, main {
  width: 100%;
  margin: 0;
  padding: 0; 
}

This makes these sections to cover the entire width all the time, irrespective of device screen width.

We also want the section, article and aside sections to be displayed as inline blocks. So, we define them thus: 

aside, section, article {
  display: inline-block;
   box-sizing:border-box;
}

display:inline-block makes targeted block elements display as inline elements. That is, they can be displayed side by side if the browser window width can accomodate that layout.

CSS Box Model

By default the CSS treats block elements as a rectangular box with height and width. The Box Model makes the margin, border, padding and the content of a block element add to its total width and height. See the below illustration.

For instance, setting a div width to width:100px, border-width:3px, margin:10px and padding:5px makes the div occupy a space of total width 118px.

Using box-sizing:border-box alters the box model of the element so that padding and border width and height values do not add to the width or height of the element. Instead, they constrain the content of the element itself. See the figure below.

That means, if I set the div to width:100px, border-width:3px, margin:10px and padding:5px, my div will need a space with width 110px to display properly without scrolling.

Media Queries

Next we want the page to display the a two-column layout with unequal width on big screens such as tablets in landscape, laptops and desktops . The smaller column to occupy 40% of the screen and the bigger column 60%. In smaller devices like phones the width of the page will just be one column.

Now to accomplish what we are saying, we use media queries. In this case our media query rule will be @media (min-width:900px){ }. This rule will query the browser screen width at the time of page rendering and whether it is 900px. If the width of the browser is 900px and above the browser will apply the rules defined within the braces { }. One of the rules we have defined in the braces is the float property. The values of float property can be none, left or right.

Here we want the smaller column to be displayed on the left and the bigger column on the right. This is how the rules will look like: 

aside {
  width: 100%;
  float: none; 
}
@media (min-width: 900px) {
    aside {
        width: 40%;
        float: left;
        margin: 0;
        padding: 10px; 
    } 
}
section {
    width: 100%; 
}
@media (min-width: 900px) {
    section {
        width: 60%;
        float: right;
        margin: 0;
        padding: 10px; 
    } 
}

One of the problem with using float property is that it shifts the target element out of the natural flow of the page content. The non-floated elements moves in to occupy the space left by a floated element, causing overlaps. If you refresh the page now, you will see that the sections are floated but the footer is not at bottom as it should be.

To solve that problem we use clear property. We define a class .group to target the floated sections. This is what we mean: 

.group:before {
    content: '';
    display: table; 
}
.group:after {
    content: '';
    display: table;
    clear: both; 
}

So what's going on? The .group class is selecting the HTML main, section and aside elements we had earlier marked with class="group". The :before and :after are pseudo selectors. Here we are inserting and empty content before and after the targeted element and display it in tabular format. But we are also clearing both the left and right content after the target element.

Possible values of clear property are left; which clears elements on the left of the floated element, right; clears right elements and both, which clears elements on the left and right of the floated element.

If you refresh the page now, the footer would have gone to its rightful place.

Styling Navigation Menu

Now it's time to style the navigation links. First we select the <nav> element. Then make the text white with color:#fff and aligned at the center with a line height of 2em. We then select the <ul> within the <nav> element and remove the bullet markers from the <ul>.

Next, instead of displaying the list vertically, we want to it displayed horizontally. We do that by using display:inline-block. We also add some padding and line height for li elements. The styles should look like the ones below. 

nav {
  color: #fff;
  text-align: center;
  height: 2em; 
}
nav ul {
    list-style: none;
    padding: .5em 0;
    line-height: 1; 
}
nav ul li {
        display: inline-block;
        padding: 0 1em; 
}

Refresh the browser and see how the page looks like.

Next we style the <a> element. We remove the default underline and change the color of the anchor to white by using text-decoration:none and color:#fff respectively. We also want when the menu item is hovered the color of the item change to shade of black and font weight to bold. This is how the styles are defined.

a {
  text-decoration: none;
  color: #fff; 
}
a:hover {
    text-weight: bold;
    color: #0a140a; 
}

We will now style the h1 heading with font-family, color, size, text-align and padding. I chose font-family aclonica. You might not have it in your system but you can chose the one you prefer. 

h1{
    color: #000;
    font-family: aclonica;
    font-size: 2em;
    text-align: center;
    padding-top: .5em;
    padding-bottom: 0; 
}

We do the same for the text Jewel of the Jungle.

.slogan{
    font-family: Allura;
    font-size: 2em;
    text-align: center;
    padding: 0; 
}

The styles above are targeting the paragraph with the class="slogan" in the document.

When chosing fonts for a webpage, it is recommended to use safe web fonts. These are fonts that are common in many client systems. However, sometimes you might want to use a unique font in your webpage and want your to viewers get the same experience whether their systems have the font or not. In such cases you need to ship the font(s) with the CSS file using @font-face rule, then specify the path where it's located in your server. But that's for another day.
